## 🛠️ JavaScript Naming Conventions Covered

| Convention | Example | Usage |
|:---|:---|:---|
| **camelCase** | `firstName`, `isUserLoggedIn` | Variables, functions (most common in JS) |
| **snake_case** | `first_name`, `is_user_logged_in` | Variables, functions (alternative style) |
| **PascalCase** | `FirstName`, `UserProfile` | Classes, constructors |
| **SCREAMING_SNAKE_CASE** | `MAX_SIZE`, `API_BASE_URL` | Constants |
| **Hungarian Notation** | `sName`, `nAge`, `bIsActive` | Type-prefixed variables (older convention) |

## 📖 Identifier Rules Summary

- Can contain letters, digits, underscores (`_`), and dollar signs (`$`)
- Must begin with a letter, underscore, or dollar sign
- **Cannot** begin with a digit
- Case-sensitive (`myVar` ≠ `MyVar`)
- **Cannot** use reserved keywords (`var`, `function`, `class`, etc.)
- Unicode letters are allowed (`ñ`, `ü`, `变量`, `π`)
